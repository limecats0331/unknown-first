# Reference

Read `../../references/shared-workflow.md` and use `../../references/artifact-templates.md` if writing files.

## Inquiry Lens

Ask: "What reference can communicate the desired behavior better than prose?"

This stage converts vague intent into concrete examples. Prefer source code, existing components, libraries, docs, screenshots, diagrams, or known-good outputs over long verbal descriptions.

## Procedure

1. Review memory, blindspot notes, prototype notes, and interview answers.
2. Identify where references would remove ambiguity.
3. Inspect or ask for references only when they materially affect the plan.
4. Summarize the transferable semantics, structure, behavior, or quality bar.
5. Record any reference-derived decisions or constraints when useful.

If no reference is needed, do not silently skip.

Do not silently skip reference. State why references would not reduce ambiguity and ask the user to confirm before proceeding to plan-and-notes.
