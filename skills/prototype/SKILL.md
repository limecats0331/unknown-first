# Prototype

Read `../references/shared-workflow.md` and use `../references/artifact-templates.md` if writing files.

## Inquiry Lens

Ask: "What tacit preference or quality bar only becomes clear when seen?"

Do not assume blindspot-pass already resolved this. Prototype exists to surface `Unknown knowns`: things the user recognizes by reacting to concrete examples.

## First Decision

Decide whether a prototype is actually useful. Do not prototype by habit.

Prototype when it can surface `Unknown knowns`: taste, layout, copy tone, workflow feel, interaction shape, output format, or "I'll know it when I see it" criteria.

Skip when the work is already specified, follows an existing pattern, or needs policy/architecture answers rather than a mock.

## If Skipping

Say why in one sentence, optionally record it in `prototype-notes.md`, then transition:

`Skip: prototype not useful because ...; next use unknown-first:interview`

## If Prototyping

1. Keep it disposable: HTML, fake data, static mock, sketch, isolated PoC, or draft artifact.
2. Do not wire production code paths.
3. Offer 2-4 meaningfully different directions when choice is the point.
4. Ask the user to react to concrete differences.
5. Record decisions and remaining unknowns in `prototype-notes.md` when useful.

## End State

Always end with:

`Next: use unknown-first:interview to resolve remaining decisions.`
